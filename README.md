## （1）制作アプリのタイトル
LTイベントリアクションシステム

## （2）制作アプリの説明（40文字程度）
LTイベント中に参加者がコメント・質問・アンケート回答を投稿し、会場スクリーンにリアルタイム表示するWebアプリケーション。

## （3）工夫した点・こだわった点
Firebaseを使いつつも、アンケート投票の回答をイベント参加者が1回だけ回答するようにするためにLocal Storageを使うなどしている。
イベントでの使用を体験を考えて、コメント投稿用のQRを表示させたり、アンケート募集中に気がつくように会場スクリーン画面にバナーを表示させたりしてる。

## （4）難しかった点・次回トライしたいこと（または機能）
自分1人でしか、確認ができなかったことが難しかった。
複数人でコメントやアンケート回答をして確認したかった。
トライしたいこととしては、イベント参加者のコメント投稿欄にもコメント一覧を表示させて「いいね」ボタンを実装したい。

## （5）使用技術
| カテゴリ | 技術・ツール |
|---|---|
| フロントエンド | HTML / CSS / Vanilla JavaScript |
| データベース | Cloud Firestore |
| 認証 | Firebase Authentication（メール＋パスワード） |
| Firebase SDK | v12.14.0 compat モード（CDN経由、npm不使用） |
| QRコード生成 | qrcodejs（CDN経由） |
| 一時保存 | LocalStorage（アンケート回答済み管理） |

## （７）動作フロー

### イベント開始前
1. 管理者が `admin.html` でログイン
2. アンケートを事前に作成しておく(`isActive: false` のまま)
3. `display.html` を会場スクリーンに投影

### イベント中(コメント収集)
1. 参加者が `index.html` でコメント投稿
2. `display.html` にリアルタイム表示

### イベント中(アンケート実施)
1. 管理画面でアンケートを「公開する」
2. 管理画面で表示モードを「アンケート結果表示」に切替
3. 参加者の画面にアンケートが出現
4. 参加者が回答 → 会場スクリーンに集計結果がリアルタイム反映
